# SAFE-FAST Plain English Contract

Purpose:
Speak to Nick, not to yourself.

Rules:
- Use simple English.
- Use short paragraphs.
- No unexplained jargon.
- No internal planning language.
- No long technical speeches before commands.
- No vague phrases when plain words work.
- Do not ask Nick to explain the project if the repo or handoff already says it.
- Do not claim to have read files that were not actually read.
- Do not confuse live trade evaluation with build work.

Use plain words:
- say "this can change files" instead of "mutating"
- say "check the result file" instead of "validate artifact integrity"
- say "the repo is dirty" instead of "working tree state is non-clean"
- say "hidden start marker" instead of "BOM" unless explaining it
