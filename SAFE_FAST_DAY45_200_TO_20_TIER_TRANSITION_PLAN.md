# SAFE-FAST $200 To $20 Tier Transition Plan

## Purpose

The next $200 month is the final high-intensity build sprint before moving toward the $20 tier. It must create a clear decision package, not an indefinite continuation loop.

## Finish During The Final High-Intensity Month

- grouped trade-plan readiness work
- grouped candidate comparison
- accepted entry rule package
- accepted exit rule package
- accepted stop/invalidation rule package
- accepted cost/slippage rule package
- grouped failure/no-trade diagnosis
- budget and remaining data-cost summary
- backtest-prep or backtest batch only after rules, data scope, and user approval exist
- decision package covering strongest paths, weakest paths, repairs, and handoff status

## Safe $20-Tier Handoff

The $20 tier can safely handle slower follow-up when the repo already has:

- organized candidate packets
- accepted rule indexes
- clear blocker lists
- focused regression fixtures
- grouped comparison summaries
- explicit next tasks
- no hidden requirement to rediscover old context

## Cannot Be Left Vague

- entry rule
- exit rule
- stop/invalidation rule
- fill assumption
- cost and slippage assumptions
- failure/no-trade labels
- sample-size expectations
- promotion and readiness gates
- remaining data costs
- which candidate families are strongest and weakest

## Budget Decisions Before More Data Spending

- Cheap starter data is used first.
- Full-window Databento data requires exact cost check and user approval.
- Broad downloads from guesses are not allowed.
- The known SPY 3-candidate full-window warning cost was about `$72.36`.
- Any serious new spend must be tied to a grouped rule, comparison, or backtest-prep batch with a clear expected use.

## Final Sprint Success Output

The final sprint succeeds if it produces a decision package that says:

- what works
- what failed
- what needs repair
- what remains blocked by data or rule gaps
- what data costs remain
- which candidate families are strongest
- which candidate families are weakest
- which rules are accepted
- which rules are still missing
- what can continue on the $20 tier
- what would require another serious spend or redesign

Success does not require forcing a weak plan to pass. It requires an honest, evidence-backed decision package and a repo that can continue at lower intensity without losing context.
